Expanding RHEL VM Volumes
1. Open Vsphere
2. Find the VM and select Edit Settings
3. Select the hard disk you would like to extend
4. Connect to the command line of the Linux VM via the console or putty session
5. Log in as your E Number
6. Type "Sudo su"
	This will put you into a root environment but it will ask you for your user password instead of the root password (once sudo has given you root privileges, su - can be executed with no password)., 
7. Type "fdisk -l"
	The fdisk command provides disk partitioning functions and using it with the -l switch lists information about your disk partitions. The response should say something like Disk /dev/sda : xxGB.
8. Verify PreScan Environment with the following:
	vgs
	Lvs
	Pvs
9. Type "echo 1 > /sys/block/sdb/device/rescan"
	We have to send “1” to /sys/class/block/XYZ/device/rescan to respective SCSI block device to refresh device information like size. So this will be helpful here since our device is already identified by kernel but we want kernel to re-read its new size and update itself accordingly.
10. Type "fdisk -l", Verify Post-Scan Environment
11. Type "pvresize /dev/sdb"
	To Resize (extend) the Physical Volume to utilize all available space on the device
12. Verify Post-Resize Environment with the following:
	fdisk -l
	vgs
	lvs
	pvs
	13. By typing "lvextend -L+[freespace]G -r /dev/volgroup/volume"
	Extend the logical volume by the amount of free space shown in the previous step
	example "lvextend -L +750G -r /dev/mapper/02opt_splunkvg-opt_splunklv"
14. Type "df -h"
	Verify the Volume is expanded
